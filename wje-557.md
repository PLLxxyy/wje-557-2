## 项目主要需求

### 项目概述

MindWell 是一款面向个人的心理健康与情绪管理平台，采用移动端优先设计，帮助用户记录情绪变化、进行冥想练习、完成心理测评，并通过数据可视化洞察情绪规律。所有数据存储在本地（localStorage/IndexedDB），绝不上传至任何服务器，充分保护用户隐私。

### 核心数据模型

#### 1. 情绪日记 MoodEntry

```typescript
interface MoodEntry {
  id: string;              // UUID
  date: string;            // ISO日期 YYYY-MM-DD
  mood: 1 | 2 | 3 | 4 | 5; // 1=很差 2=较差 3=一般 4=较好 5=很好
  emotions: EmotionTag[];  // 情绪标签，可多选
  triggers: string[];      // 触发因素
  note: string;            // 日记正文
  activities: string[];    // 当日活动
  sleepHours: number;      // 睡眠时长(小时)
  createdAt: string;
  updatedAt: string;
}
```

业务动作：
- **记录**：用户选择日期后记录当日心情等级（1-5级 Emoji 选择）、情绪标签（可多选）、触发因素、活动记录、睡眠时长和日记正文。每次记录自动生成创建时间。
- **编辑**：修改已有的日记条目，记录更新时间。同一日期仅允许一条记录，重复记录时覆盖而非新建。
- **回顾**：按日期查看历史日记，支持日历视图跳转。
- **统计**：提供月度情绪均值、最高/最低情绪日、睡眠与情绪相关性等统计数据。

#### 2. 冥想练习 MeditationSession

```typescript
interface MeditationSession {
  id: string;
  type: MeditationType;    // 练习类型
  duration: number;        // 计划时长(分钟)
  actualDuration: number;  // 实际时长(分钟)
  completedAt: string;     // 完成时间
  rating: 1 | 2 | 3 | 4 | 5; // 用户自评
  notes: string;           // 练习笔记
  interrupted: boolean;    // 是否中途退出
}
```

业务动作：
- **开始**：选择练习类型（呼吸练习/身体扫描/正念冥想/慈心冥想）和时长后开始计时。计时器页面播放柔和动画和音效提示。
- **完成**：计时结束后标记完成，引导用户记录感受评分和简短笔记。
- **回顾**：查看练习历史，包括累计练习次数、总时长、连续打卡天数、各类型分布。

#### 3. 心理测评 Assessment

```typescript
interface Assessment {
  id: string;
  type: AssessmentType;         // PHQ9/GAD7/CUSTOM
  questions: AssessmentQuestion[]; // 题目快照
  answers: number[];             // 每题得分
  totalScore: number;            // 总分
  severity: SeverityLevel;       // 正常/轻度/中度/重度
  completedAt: string;
  recommendations: string[];     // 基于结果的建议
}
```

业务动作：
- **答题**：选择量表类型后逐题作答，支持前后翻页，最后汇总确认。
- **生成报告**：根据评分标准自动计算等级并生成文字报告，包含雷达图和文字解读。
- **查看历史**：对比多次测评结果的趋势变化图。

#### 4. 目标设定 WellnessGoal

```typescript
interface WellnessGoal {
  id: string;
  category: 'sleep' | 'exercise' | 'social' | 'meditation';
  title: string;
  target: number;       // 目标值
  current: number;      // 当前值
  unit: string;         // 单位(小时/次/分钟)
  deadline: string;     // 截止日期
  status: 'active' | 'completed' | 'expired';
  createdAt: string;
}
```

业务动作：
- 创建/编辑/删除目标，系统自动计算完成进度百分比。
- 首页展示活跃目标的进度环，完成后自动标记为 completed。

### 核心页面

#### 1. `/` 今日心情（首页）

- **布局**：顶部显示当前日期和一句鼓励语（每日随机），中部展示心情选择区域（5个 Emoji 竖排排列），下方展示今日活跃目标进度列表。
- **消费数据**：MoodEntry（读写）、WellnessGoal（读取）。
- **交互**：点击 Emoji 快速记录心情后展开详细表单（情绪标签、触发因素、睡眠时长、日记正文）；若今日已有记录则显示编辑模式。
- **组件依赖**：`MoodEmoji`（心情选择器）、`ProgressRing`（目标进度）、`WeeklyCalendar`（本周概览）、`EmptyState`（无数据提示）。

#### 2. `/journal` 情绪日记

- **布局**：顶部月历热力图（日期格子颜色深浅对应心情等级），中部月度情绪趋势折线图（ECharts），下方日记条目列表。
- **消费数据**：MoodEntry（读取/筛选）。
- **交互**：点击日历日期查看当日详情；左右滑动切换月份；支持按情绪标签、心情等级筛选。
- **组件依赖**：`MoodEmoji`、`WeeklyCalendar`。
- **图表**：月度情绪趋势折线图（X轴日期，Y轴1-5分）、情绪标签词频统计饼图。

#### 3. `/meditation` 冥想练习

- **布局**：上方类型选择卡片（4种类型各配图标和简介），中部计时器（圆形进度环+倒计时数字），下方历史统计卡片。
- **消费数据**：MeditationSession（读写）。
- **交互**：选择类型和时长后进入沉浸模式（全屏计时器+呼吸动画）；计时过程中可暂停/继续；提前结束时标记为中断。
- **组件依赖**：`ProgressRing`（计时器和统计数据）、`EmptyState`。
- **图表**：本周练习时长柱状图、累计统计（总次数/总时长/连续天数）。

#### 4. `/assessment` 心理测评

- **布局**：量表选择卡片列表 → 答题页面（一题一页，底部上/下题按钮） → 结果报告页。
- **消费数据**：Assessment（读写）。
- **交互**：PHQ-9 和 GAD-7 为标准量表，按规范分值和评级标准计算；支持查看历史测评趋势对比图。
- **组件依赖**：`EmptyState`。
- **图表**：结果雷达图（多维度得分）、历史趋势折线图。

#### 5. `/insights` 数据洞察

- **布局**：分区块展示不同维度的分析图表，每个区块带标题和简要解读。
- **消费数据**：MoodEntry + MeditationSession + Assessment（综合分析）。
- **分析维度**：
  - 情绪与睡眠相关性散点图
  - 触发因素频率排行柱状图
  - 本周/本月情绪概览报告
  - 冥想练习对情绪的影响对比
- **组件依赖**：`MoodEmoji`（图例）、`ProgressRing`（完成率）。
- **图表**：ECharts 散点图、柱状图、饼图、折线图，至少4种图表类型。

#### 6. `/settings` 设置

- 主题切换（柔和暖色/冷静蓝/暗色三套主题）
- 数据管理：导出所有数据为 JSON 文件、导入数据、清除所有数据（需二次确认）

### 全局异常处理

由于本项目为纯前端应用，不涉及后端 API，因此"全局异常处理"体现为：
- **IndexedDB 操作异常捕获**：db/ 层所有数据库操作统一 try-catch，捕获后通过全局 Toast 通知用户。
- **全局错误边界**：App.vue 中注册 `app.config.errorHandler`，捕获未处理的 Vue 组件错误，记录到控制台并展示友好提示。
- **数据导入校验**：导入 JSON 数据时校验格式，非法数据弹出错误提示而非崩溃。

### 操作日志

- 所有数据写操作（创建/更新/删除 MoodEntry、MeditationSession、Assessment、WellnessGoal）记录到 IndexedDB 的 `operationLogs` 表。
- 日志字段：`{ id, action: 'CREATE'|'UPDATE'|'DELETE', entity, entityId, timestamp, snapshot }`。
- db/ 层提供 `logOperation()` 工具函数，各 store 的写操作中调用。
- 设置页提供查看操作日志的入口（开发调试用）。

### 代码严禁合并职责到单一文件

- 每个页面是一个 `.vue` 文件，不包含业务逻辑代码（逻辑抽到 hooks/）。
- 每个 store 只管理一个实体的状态，不混合其他实体逻辑。
- 图表配置（ECharts option）单独放在 `charts/` 目录，不在页面组件中内联。
- hooks/ 中每个 hook 只关注一个业务领域。
- 常量文件（enums、情绪列表、测评题目）与业务代码分离。

### 核心实体数据流

每个核心实体必须贯穿完整数据流：

```
IndexedDB → Repository(db/) → Store(stores/) → Hook(hooks/) → 页面(pages/)
                                      ↓
                              类型定义(types/) ← 枚举常量(constants/)
```

以 MoodEntry 为例：
1. `src/constants/enums.ts` 定义 `MoodLevel`、`EmotionTag`
2. `src/types/mood.d.ts` 定义 `MoodEntry` 接口，引用枚举
3. `src/db/moodRepository.ts` 实现 IndexedDB 的 CRUD 操作
4. `src/stores/moodStore.ts` 使用 Pinia 管理状态，调用 repository
5. `src/hooks/useMood.ts` 封装业务逻辑，调用 store
6. `src/pages/Journal.vue` 使用 `useMood()` hook，渲染 UI

## 代码存放规则

这是前后端分离项目。

前端代码放在 `frontend/`：
页面，组件，路由,状态管理,API 请求,类型定义,样式和资源等放在对应的子文件夹

后端代码放在 `backend/`：
数据库脚本放在 `database/`。
